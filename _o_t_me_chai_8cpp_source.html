00183 // Note optimization:
00184 // Until the actual C++ high-level API is written, we'll just use this
00185 // as a C++ wrapper to the existing high-level API written in OT Script.
00186 // Tha way, all the other languages can wrap this using swig.
00187 //
00188 // Notice (for those languages) that if you make an instance of OT_ME
00189 // and keep it around (using it for all your server calls) notice that
00190 // it will only have to instantiate and setup the OTScript object ONCE,
00191 // instead of having to do it for every single call.
00192 //
00193 // Therefore, for optimization purposes, we recommend the programmatic
00194 // user to keep an instance of OT_ME around as long as you need to use it,
00195 // and NOT instantiate it for every time you need to use it, as it will
00196 // probably run noticeably faster.
