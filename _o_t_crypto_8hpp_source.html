00548 /*
00549  ftp://ftp.rsasecurity.com/pub/pkcs/pkcs-5v2/pkcs5v2_1.pdf
00550 
00551  4.2 Iteration count
00552 
00553  An iteration count has traditionally served the purpose of increasing
00554  the cost of producing keys from a password, thereby also increasing
00555  the difficulty of attack. For the methods in this document, a minimum
00556  of 1000 iterations is recommended. This will increase the cost of
00557  exhaustive search for passwords significantly, without a noticeable
00558  impact in the cost of deriving individual keys.
00559 
00560  Time the KDF on your systems and see how many iterations are practical
00561  without signficant resource impact on legitimate use cases. If it is
00562  practical to make the count configurable, do so, otherwise hard-code
00563  a sensible value that is at least 1000.
00564 
00565  The iteration count is a multiplier on the CPU cost of brute-force
00566  dictionary attacks. If you are not sure that users are choosing "strong"
00567  passwords (they rarely do), you want to make dictionary attacks difficult
00568  by making individual password->key calculations sufficiently slow thereby
00569  limiting the throughput of brute-force attacks.
00570 
00571  If no legitimate system is computing multiple password-based keys per
00572  second, you could set the iteration count to consume 10-100ms of CPU
00573  on 2008 processors, this is likely much more than 1000 iterations.
00574  At a guess 1000 iterations will be O(1ms) per key on a typical modern CPU.
00575  This is based on "openssl speed sha1" reporting:
00576 
00577  type             16 bytes     64 bytes    256 bytes   1024 bytes   8192 bytes
00578  sha1             18701.67k    49726.06k   104600.90k   141349.84k   157502.27k
00579 
00580  or about 10^6 16-byte SHA1 ops per second, doing 1000 iterations of HMAC
00581  is approximately 2000 SHA1 ops and so should take about 2ms. In many
00582  applications 10,000 or even 100,000 may be practical provided no
00583  legitimate "actor" needs to perform password->key comptutations at
00584  a moderately high rate.
00585 
00586  */
