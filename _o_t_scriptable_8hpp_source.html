00154     // While calculating the ID of smart contracts (and presumably other scriptables)
00155     // we remove specifics such as asset types, asset accounts, Nym IDs, stashes, etc.
00156     // We override OTContract::CalculateContractID(), where we set m_bCalculatingID to
00157     // true (it's normally false). Then we call UpdateContents(), which knows to produce
00158     // an empty version of the contract if m_bCalculatingID is true. Then we hash that
00159     // in order to get the contract ID, and then we set m_bCalculatingID back to false
00160     // again.
00161     //
00162     // HOWEVER, there may be more options than with baskets (which also use the above
00163     // trick. Should the smart contract specify a specific asset type, or should it leave
00164     // the asset type blank?  Should it specify certain parties, or should it leave the
00165     // parties blank? I think the accounts should always be blank for calculating ID.
00166     // And the agents should. And stashes which should be blank in new contracts (always.)
00167     // But for asset types and parties, shouldn't people be able to specify, for a smart
00168     // contract template, whether the asset types are part of the contract or whether they
00169     // are left blank?
00170     // Furthermore, doesn't this mean that variables need to ALWAYS store their INITIAL
00171     // value, since they can change over time? We DO want to figure the variable's initial
00172     // value into the contract ID, but we do NOT want to figure the variable's CURRENT value
00173     // into that ID (because then comparing the IDs will fail once the variables change.)
00174     //
00175     // Therefore, there needs to be a variable on the scriptable itself which determines
00176     // the template type of the scriptable: m_bSpecifyAssetID and m_bSpecifyParties, which
00177     // must each be saved individually on OTScriptable.
00178     //
00179     // Agents should be entirely removed during contract ID calculating process, since
00180     // the Parties can already be specified, and since they can choose their agents at the
00181     // time of signing -- which are otherwise irrelevant since only the parties are liable.
00182     //
00183     // Accounts, conversely, CAN exist on the contract while calculating its ID, but the
00184     // actual account IDs will be left blank and the asset type IDs will be left blank
00185     // if m_bSpecifyAssetID is false. (Just as Parties' Owner IDs will be left blank
00186     // if m_bSpecifyParties is false.)
00187     //
00188     // Transaction numbers on parties AND accounts should be set to 0 during calculation
00189     // of contract ID. Agent name should be left blank on both of those as well.
00190     //
00191     // On OTParty, signed copy can be excluded. All agents can be excluded. Authorizing agent
00192     // can be excluded and Owner ID is conditional on m_bSpecifyParties. (Party name is kept.)
00193     // m_bPartyIsNym is conditional and so is m_lOpeningTransNo.
00194     //
