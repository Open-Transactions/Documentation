00143 // This is a good implementation. Dots all the i's, so to speak.
00144 // client-side.
00145 // The basket ONLY stores closing numbers, so this means "harvest 'em all."
00146 //
00147 // NOTE: The basket might be harvested in different ways, depending on context:
00148 //
00149 // 1. If the command-line client (OR ANY OTHER CLIENT) has a failure BEFORE sending the message,
00150 //    (e.g. while constructing the basket exchange request), then it should call OTAPI.Msg_HarvestTransactionNumbers
00151 //    and pass in the exchange basket string. That function will check to see if the input is an
00152 //    exchange basket, and if so, it will load it up (AS A BASKET) into OTBasket and call the below
00153 //    function to harvest the numbers.
00154 //
00155 // 2. If the high-level API actually SENDS the message, but the message FAILED before getting a chance
00156 //    to process the exchangeBasket transaction, then the high-level API will pass the failed message
00157 //    to OTAPI.Msg_HarvestTransactionNumbers, which will load it up (AS A MESSAGE) and that will then
00158 //    call pMsg->HarvestTransactionNumbers, which then loads up the transaction itself in order to call
00159 //    pTransaction->HarvestClosingNumbers. That function, if the transaction is indeed an exchangeBasket,
00160 //    will then call the below function OTBasket::HarvestClosingNumbers.
00161 //
00162 // 3. If the high-level API sends the message, and it SUCCEEDS, but the exchangeBasket transaction inside
00163 //    it has FAILED, then OTClient will harvest the transaction numbers when it receives the server reply
00164 //    containing the failed transaction, by calling the below function, OTBasket::HarvestClosingNumbers.
00165 //
00166 // 4. If the basket exchange request is constructed successfully, and then the message processes at the server
00167 //    successfully, and the transaction inside that message also processed successfully, then no harvesting will
00168 //    be performed at all (obviously.)
00169 //

00179     
00180     for (uint32_t i = 0; i < nCount; i++)
00181     {

00195 //      else
