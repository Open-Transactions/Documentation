00164 // Why does the map of mints use multimap instead of map?
00165 // Because there might be multiple valid mints for the same asset type.
00166 // Perhaps I am redeeming tokens from the previous series, which have not yet expired.
00167 // Only tokens from the new series are being issued today, but tokens from the previous
00168 // series are still good until their own expiration date, which is coming up soon.
00169 // Therefore the server manages different mints for the same asset type, and since the asset
00170 // type is the key in the multimap, we don't want to accidentally remove one from the list
00171 // every time another is added. Thus multimap is employed.
