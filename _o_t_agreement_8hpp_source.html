00275     // This function verifies both Nyms and both signatures.
00276     // Due to the peculiar nature of how OTAgreement/OTPaymentPlan works, there are two signed
00277     // copies stored. The merchant signs first, adding his transaction numbers (2), and then he
00278     // sends it to the customer, who also adds two numbers and signs. (Also resetting the creation date.)
00279     // The problem is, adding the additional transaction numbers invalidates the first (merchant's)
00280     // signature.

00283     // and BOTH Nyms can be checked to verify that BOTH transaction numbers are valid for each.
